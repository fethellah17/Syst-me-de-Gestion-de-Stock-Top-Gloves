| Status Badge (Case B) | 🟠 Orange | Partial Acceptance - PARTIELLEMENT ACCEPTÉ |
| Status Badge (Case C) | 🔴 Red | Total Refusal - REFUSÉ |

## Key Differences

### Header
- **Case A & B:** "Bon d'Entree"
- **Case C:** "BON DE REFUS DE RECEPTION" (Red, prominent)

### Status Badge
- **Case A:** Green "CONFORME"
- **Case B:** Orange "PARTIELLEMENT ACCEPTÉ"
- **Case C:** Red "REFUSÉ"

### Quantity Display
- **Case A:** Single line with 100% acceptance
- **Case B:** Professional table with 3 columns (Reçue, Acceptée, Rejetée)
- **Case C:** Zero acceptance with "REFUS TOTAL"

### Observations
- **Case A:** Not shown (no defects)
- **Case B:** Shown with control notes
- **Case C:** Shown as "MOTIF DU REFUS" in red box

### Signature Section
- **All Cases:** Same format with Operator, QC Controller, and Date

## Data Mapping

### From QC Modal to PDF

| QC Modal Field | PDF Usage | Case A | Case B | Case C |
|---|---|---|---|---|
| `controleur` | QC Controller name | ✓ | ✓ | ✓ |
| `qteValide` | Accepted quantity | ✓ | ✓ | 0 |
| `qteDefectueuse` | Defective quantity | 0 | ✓ | Total |
| `noteControle` | Observations | — | ✓ | — |
| `refusTotalMotif` | Refusal reason | — | — | ✓ |

## User Actions

### To Generate PDF:
1. Create Entrée movement
2. Complete QC inspection in modal
3. Approve or reject
4. Click PDF button in movement table
5. PDF downloads automatically

### PDF Filename:
- **Case A & B:** `Bon_Entree_[MOVEMENT_ID].pdf`
- **Case C:** `Bon_Refus_Reception_[MOVEMENT_ID].pdf`

## Testing Scenarios

### Scenario 1: Perfect Delivery
- Receive 500 units
- QC: 0 defects
- Result: Case A PDF (Green CONFORME)

### Scenario 2: Damaged Shipment
- Receive 500 units
- QC: 20 defects found
- Add notes: "Packaging damaged"
- Result: Case B PDF (Orange PARTIELLEMENT ACCEPTÉ)

### Scenario 3: Rejected Lot
- Receive 500 units
- QC: Total rejection
- Add reason: "Missing certificates"
- Result: Case C PDF (Red BON DE REFUS)
